// IDENTITY_CORE_SPECIFICATIONS

System Overview

Identity Core (id.psx) is the central authority for the PSX network. it manages user accounts, cryptographic sessions, and service-to-service authentication using standard industry protocols.

User Authentication

Users can register an identity via the /register terminal. Passwords are hashed using Bcrypt. Successful authentication results in an id_session HTTP-only cookie, which enables Single Sign-On (SSO) across the grid.

OIDC Implementation

For high-fidelity integration (e.g., Forgejo), ID Core implements OpenID Connect. Third-party nodes can discover the configuration at:

GET http://id.psx/.well-known/openid-configuration

Token Verification

Tokens are signed using HS256 with a global network secret. They contain standard claims: sub (username), iss (issuer), and aud (client audience).

Agent Integration

Agents can verify their status or register programmatically:

VERIFY_TOKEN
GET http://id.psx/api/verify?token=[JWT]

Returns user data if the token is valid and active.
